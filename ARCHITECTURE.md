## 📦 核心模块

### 1. BaseAgent (基础Agent类)

**职责**: 提供LLM调用的通用能力

**核心方法**:
- `call_llm(prompt, system_prompt)`: 调用OpenAI API

**特性**:
- 统一的API接口封装
- 支持模型切换
- 错误处理

### 2. RequirementAnalyzer (总控Agent)

**输入**: 原始需求文本

**输出**: 结构化分析结果
```json
{
  "goal": "需求目标",
  "users": "目标用户",
  "core_features": "核心功能点",
  "unclear_points": "不明确点",
  "dependencies": "依赖模块"
}
```

**Prompt设计要点**:
- 角色设定：资深产品专家
- 任务明确：5个维度分析
- 格式约束：JSON输出

### 3. Engineering Agent (研发Agent)

**关注点**:
- 功能模块拆解
- API接口设计
- 异常处理
- 技术边界

**Prompt策略**:
```
角色: 资深技术负责人
风格: 专业、具体、可执行
结构: 7个标准章节
```

### 4. Design Agent (设计Agent)

**关注点**:
- 用户任务流程
- 页面状态设计
- 交互反馈
- 视觉层级

**Prompt策略**:
```
角色: 资深UX/UI设计师
风格: 用户中心、体验导向
结构: 7个标准章节
```

### 5. AI/Algorithm Agent (算法Agent)

**关注点**:
- AI能力定义
- Prompt工程
- 评测指标
- Fallback策略

**Prompt策略**:
```
角色: 资深AI算法工程师
风格: 技术严谨、量化指标
结构: 8个标准章节
```

### 6. QA Agent (测试Agent)

**关注点**:
- 测试用例设计
- 边界场景
- 成功/失败判定
- 自动化测试

**Prompt策略**:
```
角色: 资深QA工程师
风格: 全面、细致、可量化
结构: 10个标准章节
```

### 7. Operation Agent (运营Agent)

**关注点**:
- 用户价值
- 商业价值
- 卖点话术
- 关键指标

**Prompt策略**:
```
角色: 资深运营专家
风格: 业务导向、数据驱动
结构: 10个标准章节
```

### 8. RiskAnalyzer (风险分析Agent)

**输入**: 
- 原始需求
- 分析结果
- 各部门转译版本

**输出**: 风险评估报告
- 不明确表达
- 需补充信息
- 跨部门偏差
- 隐性依赖
- 可行性风险
- 下一步建议

**Prompt策略**:
```
角色: 资深项目管理专家
风格: 批判性思维、建设性意见
方法: 综合分析所有输入
```

## 🔄 工作流程

```
1. 用户输入原始需求
         ↓
2. RequirementAnalyzer 解析
         ↓
   结构化分析结果
         ↓
3. 并行调用5个部门Agent
   ├─→ Engineering Agent
   ├─→ Design Agent
   ├─→ AI Agent
   ├─→ QA Agent
   └─→ Operation Agent
         ↓
   5个部门专属版本
         ↓
4. RiskAnalyzer 综合分析
         ↓
   风险评估 + 建议
         ↓
5. 统一展示给用户
```

## 🎯 Prompt工程实践

### 设计原则

1. **角色明确**: 每个Agent都有清晰的角色定位
2. **关注点分离**: 不同Agent关注不同维度
3. **结构约束**: 规定输出格式和章节
4. **风格指导**: 明确语言风格和表达方式
5. **示例引导**: 在prompt中给出示例（few-shot）

### Prompt模板

```python
system_prompt = """你是{角色}，擅长{专业能力}。
关注点：
- {关注点1}
- {关注点2}
- ...

输出风格：{风格描述}"""

prompt = """基于以下产品需求和分析结果，生成给{部门}的需求文档：

【原始需求】
{raw_requirement}

【需求分析】
- 目标：{goal}
- 用户：{users}
- 核心功能：{features}

请按照以下结构输出：
{结构模板}"""
```

### 优化技巧

1. **迭代优化**: 根据输出结果调整prompt
2. **A/B测试**: 对比不同prompt版本的效果
3. **温度调节**: temperature=0.7平衡创造性和稳定性
4. **长度控制**: max_tokens限制输出长度
5. **格式验证**: 后处理确保输出格式正确

## 🔧 技术选型理由

### Streamlit
- ✅ 快速构建数据应用
- ✅ Python原生，无需前端开发
- ✅ 内置组件丰富
- ✅ 自动热重载

### OpenAI GPT
- ✅ 强大的自然语言理解
- ✅ 稳定的API服务
- ✅ 多模型选择
- ✅ 良好的中文支持

### Multi-Agent架构
- ✅ 职责清晰，易于维护
- ✅ 可独立优化每个Agent
- ✅ 易于扩展新角色
- ✅ 符合真实工作流程

## 🚀 性能优化

### 当前实现
- 串行调用各部门Agent（便于观察进度）
- 每次调用独立LLM请求

### 优化方向
1. **并行调用**: 使用asyncio并发执行Agent
2. **缓存机制**: 相同需求不重复调用
3. **流式输出**: 使用streaming减少等待感
4. **批量处理**: 多个需求一起处理
5. **本地缓存**: 缓存常用响应

## 🔒 安全考虑

1. **API Key管理**: 
   - 不在代码中硬编码
   - 通过UI输入，不持久化
   
2. **输入验证**:
   - 检查空输入
   - 限制输入长度
   
3. **错误处理**:
   - API调用失败降级
   - 友好的错误提示
